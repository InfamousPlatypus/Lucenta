# Lucenta Roadmap

## Phase 1: The "Legacy Guard" (Active Phase)
**Focus:** Security, Isolation, and Migration.
**Status:** In Progress.

- **Model Provider:** Hardware-aware routing (NCS2/iGPU vs. API).
- **Security Auditor:** Static "Smell Test" for Node.js, Python, and C++.
- **OpenClaw Bridge:** Translating SKILL.md to secure MCP Tools.
- **Docker Locker:** Resource-capped sandbox for all external code.
- **Docling MCP:** Hardware-accelerated document intelligence.
- **HIL Gateway:** Telegram & CLI human-approval interface.

## Phase 2: Sensory Expansion & Metal Trust
**Focus:** Efficiency, Proactive Intelligence, and Hardware Promotion.

- **Trust Promotion:** Plugins that pass 100 successful runs without security flags can be "promoted" to run on Metal (host OS).
- **Multimodal Gateway:** Implementation of local Whisper (STT) and Kokoro (TTS) for voice-based interaction via Telegram.
- **Active Monitoring:** Lucenta starts "watching" project folders for automatic parsing (e.g., Docling).
- **Proactive Tasking:** Event-based scheduling (e.g., "Alert me if a new GitHub Issue matches my current project").

## Phase 3: Zero-Trust Autonomy
**Focus:** Identity, Collaborative AI, and Edge Sovereignty.

- **Cryptographic Identity:** Biometric HIL or signed commands via Telegram.
- **Swarm Coordination:** Multiple Lucenta instances coordinating via encrypted PGP channels.
- **Full Edge Sovereignty:** Local models (Llama 3 / Phi-4) take over 90% of Triage and Audit duties.
